Задача: Научиться работать с сортировкам, структурами, создать рекурсивные функции

возможности программы 

"Математические вычисления"
реализовано два варианта каждой функции, обычный подсчет и рекурсивный 
- Факториал - вычисление произведения натуральных чисел от 1 до заданного значения
- Числа Фибоначчи - генерация элементов известной числовой последовательности

"Алгоритмы сортировки"
- Пузырьковая сортировка - Сортирует массив, многократно проходя по нему и меняя местами соседние элементы, если они стоят в неправильном порядке.
- Быстрая сортировка - Эффективный алгоритм сортировки, который разбивает массив на части и рекурсивно сортирует каждую часть.
- Сортировка подсчетом - алгоритм для целочисленных значений, считает количество каждоко из всевозможных чисел в исходном массиве 
- Поразрядная сортировка -  Сортирует числа по разрядам (единицы, десятки, сотни и т.д.).
- Блочная сортировка - Распределяет элементы по "ведрам", сортирует каждое ведро отдельно, затем объединяет.

"Структуры данных"
- Стек - структура данных с принципом LIFO (последним пришел - первым ушел) реализовано на двусвязном списке 

*push(a) - добавить элемент на вершину

*pop() - удалить и вернуть верхний элемент

*top() - посмотреть верхний элемент

*mini() - найти минимальный элемент в стеке

- Очередь - структура данных с принципом FIFO (первым пришел - первым ушел) реализовано на двух стеках

*enqueue(a) - добавить элемент в конец

*dequeue() - удалить и вернуть первый элемент

*front() - посмотреть первый элемент

Чему научилась:

1. Любую сложную задачу можно разбить на простые шаги (рекурсия)
2. Разные задачи требуют разных подходов (для каждого массива подходит своя сортировка)
3. Эффективность/Простота (рекурсию проще понять, но она может работать медленнее)
4. Универсальность структур данных (стек и очередь можно реализовать в рахных представлениях)

Ограничения:
1. Рекурсия 
- Очень медленно для больших чисел 
- Переполнение стека при глубокой рекурсии
2. ортировка пузырьком
- Квадратичная сложность - не подходит для больших данных
- Неэффективно на частично отсортированных массивах
3. Сортировка подсчетом
- Большой расход памяти если диапазон значений широкий
- Не подходит для дробных чисел или строк
4. Поразрядная сортировка
- Только для целых чисел
- Требует дополнительной памяти для корзин
